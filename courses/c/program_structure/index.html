Program structure

Let's start off by looking at what each C program is made up of.
The significant majority C programs have a main function/method - This is where the CPU begins executing the code, or rather,
the assembly instructions resulting from your C code. Unlike some other programming languages, like Java, C is compiled directly to
machine language, which your Central Processing Unit can directly interpret and execute, yielding greater performance over interpreted languages.
There are exceptions to this rule, however. [Dynamically linked] libraries, or any program linked with another program for that matter, may have a different entry point, which is then called by the master program.

In the next chapter, we will take a closer look at methods/functions, and how to define and call them.
